# Пульс соцсетей — дашборд статистики VK и YouTube

Дашборд автоматически собирает раз в час статистику группы VK и канала
YouTube (подписчики, лайки, комментарии, репосты, просмотры) и хранит
историю для ретроспективы.

## Структура проекта

```
public/        — сайт (HTML/CSS/JS), который видит пользователь
api/collect.js — сбор данных из VK и YouTube, запускается по расписанию
api/stats.js   — отдаёт агрегированные данные дашборду
lib/           — вспомогательные функции (VK API, YouTube API, база данных)
vercel.json    — настройка расписания (раз в час)
```

## Развёртывание (один раз)

### 1. Создать базу данных

На [vercel.com](https://vercel.com) → раздел **Storage** → **Create Database** →
выбрать **Postgres** (через Neon). После создания Vercel сам добавит переменную
`DATABASE_URL` в проект — отдельно копировать не нужно.

### 2. Подключить репозиторий к Vercel

1. Зарегистрироваться/войти на vercel.com через GitHub.
2. **Add New Project** → выбрать этот репозиторий.
3. Framework Preset — **Other**.
4. Нажать **Deploy**.

### 3. Добавить переменные окружения

В настройках проекта на Vercel → **Settings → Environment Variables**
добавить:

| Имя | Значение |
|---|---|
| `VK_GROUP_TOKEN` | ключ доступа сообщества VK (из «Работа с API») |
| `VK_GROUP_ID` | ID или короткое имя группы, например `237224882` |
| `YOUTUBE_API_KEY` | API key из Google Cloud Console |
| `YOUTUBE_CHANNEL_ID` | ID канала, начинается на `UC...` |
| `COLLECT_SECRET` | любая произвольная строка-пароль для ручного запуска сборщика |

После добавления переменных — **Redeploy** проект (кнопка в разделе Deployments),
чтобы они применились.

### 4. Проверить сбор данных вручную

Открыть в браузере:
```
https://ваш-сайт.vercel.app/api/collect?secret=ВАШ_COLLECT_SECRET
```
Должен вернуться JSON с результатами сбора. Если есть ошибки — они будут
видны в поле `errors`.

### 5. Автоматический запуск

Файл `vercel.json` уже настроен на запуск `/api/collect` каждый час
(`0 * * * *`). Vercel включает Cron Jobs автоматически после деплоя —
ничего дополнительно настраивать не нужно.

## Локальная проверка (опционально)

```bash
npm install
npx vercel dev
```
